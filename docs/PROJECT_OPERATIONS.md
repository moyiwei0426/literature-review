# PROJECT OPERATIONS

## 开始任一任务前
按顺序读取：
1. `progress/STATUS.md`
2. `plans/NEXT_ACTIONS.md`
3. `progress/WORKLOG.md`
4. 当前步骤对应的 `plans/steps/` 与 `plans/details/` 文件

## 完成任一任务后
必须更新：
1. `progress/STATUS.md`
2. `progress/WORKLOG.md`
3. `plans/NEXT_ACTIONS.md`

如架构变更，再更新：
4. `docs/ARCHITECTURE.md`

## 更新建议格式
### STATUS
- 当前阶段
- 已完成
- 进行中
- 下一关键动作

### WORKLOG
- 时间
- 做了什么
- 产出文件
- 问题/风险
- 下一步

### NEXT_ACTIONS
只保留 3~5 个最关键动作，按执行顺序排列。
